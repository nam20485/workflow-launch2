ImperativeUI

UI Generation & Rendering Architecture Comparison

Option A: WebView

Render UI using HTML/CSS/JS inside a browser component.

Generator Target

HTML, CSS, JavaScript

Viewer

Avalonia `WebView` Control

Benefits

  • Ultimate UI flexibility & styling (CSS)
  • Leverages existing web skills & libraries
  • Generated UI is portable to the web

Disadvantages

  • Complex C# to JS data binding
  • Higher performance overhead
  • Inconsistent native look & feel
  • Increases application size

Option B: Native

Recommended

Render UI by programmatically creating native Avalonia controls.

Generator Target

C# Avalonia Control Instances

Viewer

Native Avalonia `Window` / Panel

Benefits

  • Optimal performance and efficiency
  • Perfect native look & feel
  • Simple, reliable, and type-safe data binding
  • Smallest application size

Disadvantages

  • Styling is less flexible than CSS
  • Complex custom controls require more effort

Option C: Dynamic XAML

Render UI by generating and parsing Avalonia's XAML markup at runtime.

Generator Target

String of XAML Markup

Viewer

Native Avalonia `ContentControl`

Benefits

  • Clean, declarative generation target
  • Good separation of UI definition and logic
  • Leverages existing XAML skills

Disadvantages

  • Runtime XAML parsing is slow
  • Potential security risks from generated code
  • Debugging can be more difficult
